    A

    A2A (Agent-to-Agent)
    Google's reference protocol for agents to discover and collaborate with peer agents across boundaries. → Ch 13
    A2UI (Agent-to-UI)
    An emerging, still-forming idea where agents generate or drive user interfaces directly. → Ch 20
    ACP (Agent Communication Protocol)
    An emerging category of protocols for peer agent messaging; wire formats are still converging. → Ch 12
    Agent
    An LLM given tools, memory, and a loop so it can act — not just answer — to reach a goal. → Ch 08
    Agent Card
    In Google's A2A, a machine-readable profile advertising an agent's identity, skills, and endpoint. → Ch 13
    Agentic AI
    Systems where agents plan, decompose goals, delegate, and coordinate with limited human input. → Ch 09
    Anti-pattern
    A common but counterproductive design choice that seems reasonable yet causes harm at scale. → Ch 04
    Artifact
    In A2A, a structured output a remote agent returns as the result of a task. → Ch 13
    Attention
    The mechanism letting a Transformer weigh how much each token should influence every other token. → Ch 04
    Autonomous enterprise
    An operating model where agents own end-to-end processes under governance and human direction. → Ch 20

    B

    Backpropagation
    The algorithm that adjusts a neural network's weights by propagating error gradients backward. → Ch 03
    Batching
    Grouping requests to share compute and cut per-call cost and latency. → Ch 16

    C

    Chain-of-thought
    Prompting a model to reason step by step before answering, improving accuracy on hard tasks. → Ch 06
    Chunking
    Splitting documents into passages so a retriever can find and return the relevant pieces. → Ch 07
    CNN (Convolutional Neural Network)
    A deep network that detects local patterns, historically strong for images. → Ch 03
    Context engineering
    Deliberately assembling the right information into the model's finite context window. → Ch 14
    Context ontology
    Enterprise knowledge modeling of entities and relationships to keep context coherent (used conservatively). → Ch 14
    Context window
    The maximum number of tokens a model can attend to in a single request. → Ch 05

    D

    Decision tree (design)
    A branching guide for choosing an approach based on the situation's constraints. → Ch 08
    Decoder-only model
    A Transformer wired for generation via next-token prediction — the dominant LLM design. → Ch 04
    Deep learning
    Machine learning with many-layer neural networks that learn features from raw data. → Ch 03
    Distillation
    Training a smaller model to mimic a larger one, cutting cost while keeping much of the quality. → Ch 16

    E

    Egress control
    Restricting where an agent can send data, limiting exfiltration risk. → Ch 18
    Embedding
    A numeric vector representing text so that similar meanings sit close together. → Ch 07
    Encoder-decoder model
    A Transformer that encodes an input then decodes an output — used for translation and summarisation. → Ch 04
    Eval (evaluation)
    A repeatable test that scores model or agent behaviour against expected outcomes. → Ch 17
    Excessive agency
    Giving an agent more capability or permission than its task requires — a key security risk. → Ch 18

    F

    Few-shot prompting
    Including a handful of examples in the prompt to steer the model's output format and behaviour. → Ch 06
    Fine-tuning
    Further training a model on domain data to adjust its weights for a specific use. → Ch 05

    G

    Golden dataset
    A curated set of inputs with known-good outputs used as the backbone of offline evaluation. → Ch 17
    Groundedness
    The degree to which a generated answer is supported by the retrieved source material. → Ch 17
    Guardrails
    Layered controls that validate inputs, outputs, and actions to keep agents safe and on-policy. → Ch 15

    H

    Hallucination
    A confident but false or unsupported model output. → Ch 07
    Human-in-the-loop
    Requiring human approval for high-stakes or irreversible agent actions. → Ch 15

    I

    Indirect prompt injection
    Malicious instructions hidden in data an agent reads (e.g. a web page) rather than the user's message. → Ch 18
    Inference
    Running a trained model to produce an output for a given input. → Ch 05

    J

    JSON-RPC
    A lightweight remote-procedure-call format used by MCP and A2A over HTTP. → Ch 11

    L

    LangGraph
    A framework for building stateful, multi-step agent workflows as graphs. → Ch 10
    Large Language Model (LLM)
    A large Transformer trained on vast text to become a general-purpose language platform. → Ch 05
    Latency
    The time taken to produce a response — a core cost and UX constraint. → Ch 16
    Least privilege
    Granting an agent only the minimum permissions and tools it needs. → Ch 18
    LLM-as-judge
    Using an LLM to score another model's output against a rubric during evaluation. → Ch 17

    M

    Machine learning
    Systems that learn patterns from data instead of following hand-written rules. → Ch 02
    MCP (Model Context Protocol)
    A standard client/server protocol that connects models to tools, data, and prompts. → Ch 11
    Mermaid
    A text-to-diagram tool used throughout this handbook to render flow and state diagrams. → Ch 04
    Model routing / tiering
    Sending each request to the cheapest model that can meet the quality bar. → Ch 16
    Multi-agent system
    Several specialised agents collaborating, often via a supervisor or peer protocols. → Ch 09

    N

    Neural network
    Layers of interconnected weighted units that transform inputs into outputs. → Ch 03

    O

    Offline evaluation
    Scoring a system against a fixed dataset before deployment. → Ch 17
    Online evaluation
    Measuring real production behaviour via tracing, metrics, and A/B tests. → Ch 17
    Orchestration
    Coordinating multiple steps, tools, or agents into a reliable stateful workflow. → Ch 10
    Overfitting
    When a model memorises training data and fails to generalise to new inputs. → Ch 02

    P

    Prompt
    The input text and instructions given to an LLM to elicit a response. → Ch 06
    Prompt caching
    Reusing computation for repeated prompt prefixes to cut cost and latency. → Ch 16
    Prompt engineering
    The practice of crafting prompts to steer an LLM toward reliable, useful outputs. → Ch 06
    Prompt injection
    An attack that smuggles instructions into a prompt to override intended behaviour. → Ch 18

    Q

    Quantisation
    Reducing the numeric precision of model weights to shrink cost and memory. → Ch 16

    R

    RAG (Retrieval-Augmented Generation)
    Grounding an LLM by retrieving relevant documents and adding them to the prompt. → Ch 07
    Red-teaming
    Deliberately attacking a system to surface safety and security failures before attackers do. → Ch 18
    Reranking
    Reordering retrieved candidates by relevance before passing them to the model. → Ch 07
    RNN / LSTM
    Recurrent networks that process sequences step by step; largely superseded by Transformers. → Ch 04

    S

    Scaling laws
    The empirical trend that model quality improves predictably with more data, parameters, and compute. → Ch 05
    Self-attention
    Attention applied within a single sequence so each token can look at all others. → Ch 04
    Server-Sent Events (SSE)
    A streaming transport A2A uses to push incremental task updates over HTTP. → Ch 13
    System prompt
    Standing instructions that define an assistant's role, rules, and tone. → Ch 06

    T

    Temperature
    A sampling setting controlling randomness in an LLM's output. → Ch 06
    Token
    The sub-word unit LLMs read and generate; billing and context limits are measured in tokens. → Ch 05
    Token economics
    Managing cost, latency, and context budgets driven by input and output token counts. → Ch 16
    Tool (function calling)
    An external capability an agent can invoke, such as a search or a database write. → Ch 08
    Tracing
    Recording the full path of a request through a system for debugging and online evaluation. → Ch 17
    Transformer
    The attention-based architecture underpinning modern LLMs. → Ch 04
    Trust boundary
    A line where data or control passes between differently-trusted components; a key security control. → Ch 18

    V

    Vector database
    A store that indexes embeddings for fast similarity search in RAG. → Ch 07
    Verification (AI-native)
    Confirming AI-generated changes are correct via tests, evals, and review — the new bottleneck. → Ch 19
